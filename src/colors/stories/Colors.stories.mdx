# Colors

<Subtitle>
  Consistent use of color is important to help users quickly understand information. Most colors are reserved for
  communicating specific concepts/meaning (like status) or for specific component types (like icons).
</Subtitle>

## Status

These colors are reserved for conveying specific kinds status or health.
They should only be used in that context.

### Standard

Useful as a backdrop for icons
or where a higher level of contrast with surrounding elements is desired.

<ColorPalette>
  {Object.entries(colors.status.standard).map(([name, color]) => (
    <ColorItem title={name} colors={[color]} />
  ))}
</ColorPalette>

### Light

Useful as a background color on larger elements,
or where a lower level of contrast with surrounding elements is desired.

<ColorPalette>
  {Object.entries(colors.status.light).map(([name, color]) => (
    <ColorItem title={name} colors={[color]} />
  ))}
</ColorPalette>

## Validation

These colors are reserved for conveying whether a particular state
or user input is valid/invalid, for example while interacting with forms.
They should only be used in that context.

### Standard
